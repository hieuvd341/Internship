# Heap one
Source code:
```c
#include <stdlib.h>
#include <unistd.h>
#include <string.h>
#include <stdio.h>
#include <sys/types.h>

struct internet {
  int priority;
  char *name;
};

void winner()
{
  printf("and we have a winner @ %d\n", time(NULL));
}

int main(int argc, char **argv)
{
  struct internet *i1, *i2, *i3;

  i1 = malloc(sizeof(struct internet));
  i1->priority = 1;
  i1->name = malloc(8);

  i2 = malloc(sizeof(struct internet));
  i2->priority = 2;
  i2->name = malloc(8);

  strcpy(i1->name, argv[1]);
  strcpy(i2->name, argv[2]);

  printf("and that's a wrap folks!\n");
}
```
Chương trình cấp phát bộ nhớ động cho i1 và i2, sau đó tiếp tục cấp phát bộ nhớ động cho ```i1->name``` và ```i2->name```. Lúc này bên trong heap, bên cạnh giá trị priority của i1 và i2, thay vì giá trị của ```i1->name``` hoặc ```i2->name``` sẽ là địa chỉ của ```i1->name``` và ```i2->name```. Kiểm soát được địa chỉ này và ghi địa chỉ của hàm winner và bên trong GOT ta sẽ chuyển hướng được chương trình.
